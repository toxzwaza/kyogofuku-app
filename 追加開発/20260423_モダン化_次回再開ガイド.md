## 4. デザインシステム規約（必読）

### 4.1 カラートークン

**必ずこのトークンを使う**。生の色（`bg-white`、`text-gray-800`、`bg-indigo-600` 等）を新規コードに書かない。

#### 7色スケール（各11段階: 50, 100, 200, 300, 400, 500, 600, 700, 800, 900, 950）

| トークン | 用途 | 500 | 備考 |
|---------|------|------|------|
| `sumi` (墨) | neutral / text | `#62626B` | テキスト・罫線・ニュートラル背景 |
| `ai` (藍) | primary | `#2C5C94` | 主要操作・リンク |
| `enji` (臙脂) | accent | `#A7413F` | アクセント・強調 |
| `unohana` (卯の花) | surface warm | `#C4AF6E` | 生成り系背景 |
| `uguisu` (鶯) | success | `#6A8737` | 成功・有効 |
| `natane` (菜種) | warning | `#B5920F` | 警告・返信待ち |
| `akane` (茜) | danger | `#C42929` | エラー・キャンセル |

#### brand-* セマンティックトークン（CSS変数、ダーク自動対応）

```
bg-brand-primary / bg-brand-primary-hover / text-brand-on-primary
bg-brand-accent / text-brand-on-accent
bg-brand-success / bg-brand-warning / bg-brand-danger
bg-brand-surface / bg-brand-surface-2 / bg-brand-bg
border-brand-border / border-brand-border-strong
text-brand-text / text-brand-text-muted / text-brand-text-subtle
```

**ルール**:
- **UI要素には `brand-*` を優先**（自動でダーク対応される）
- 固定色が必要な場合のみ `ai-600` 等の生トークンを使う
- `bg-white`、`text-gray-*`、`indigo-*` 直書きは **禁止**

### 4.2 タイポグラフィ

| 用途 | クラス |
|------|--------|
| セクション見出し・ブランド的な要素 | `font-serif`（Zen Old Mincho） |
| 本文・UI文字 | デフォルト sans（Figtree+Noto Sans JP） |
| コード・電話・メール等 | `font-mono` / `UiDetailField mono` |

**見出しパターン**: `<h3 class="font-serif text-base font-semibold flex items-center gap-2 text-brand-text">`

### 4.3 角丸・シャドウ

| トークン | 用途 |
|---------|------|
| `rounded-soft-sm` (4px) | 小さい要素 |
| `rounded-soft` (8px) | 標準（ボタン、Inputなど） |
| `rounded-soft-lg` (12px) | カード・Dialog |
| `rounded-soft-xl` (20px) | 大きなコンテナ |
| `shadow-soft-sm / soft / soft-lg` | 階層別の柔らかい影 |

### 4.4 アイコン

- **必ず Lucide-Vue-Next を使う**。インラインSVGは書かない。
- 標準サイズ: 小 `:size="12-14"` / 中 `:size="15-16"` / 大 `:size="18-20"`
- アイコンライブラリ: `import { Plus, Pencil, Trash2, ... } from 'lucide-vue-next'`

## 5. UIキットの使い方

### 5.1 必ず使うコンポーネント

```vue
<!-- 1. ページレイアウト -->
<AdminLayout :breadcrumb="[{ label: '顧客' }, { label: '顧客詳細' }]">
  <UiPageHeader title="顧客詳細" description="...">
    <template #actions>
      <UiButton variant="primary" :href="...">保存</UiButton>
    </template>
  </UiPageHeader>

  <!-- 2. ページ内容 -->
  <UiCard variant="default" padding="lg">
    <template #header>
      <h3 class="font-serif text-base font-semibold flex items-center gap-2 text-brand-text">
        <User :size="15" class="text-brand-primary" />
        基本情報
      </h3>
    </template>

    <!-- 3. 情報表示セクション -->
    <UiDetailSection title="プロフィール" :icon="UserCircle" :cols="2">
      <UiDetailField label="名前" :value="customer.name" :icon="User" highlight />
      <UiDetailField label="メール" :value="customer.email" :icon="Mail" mono />
    </UiDetailSection>

    <!-- 4. フォーム -->
    <UiFormField label="名前" :error="form.errors.name">
      <UiInput v-model="form.name" placeholder="例: 山田 太郎" />
    </UiFormField>
    <UiFormField label="店舗">
      <UiSelect v-model="form.shop_id" :options="shopOptions" />
    </UiFormField>

    <!-- 5. アクション -->
    <UiButton variant="primary" type="submit" :loading="form.processing">
      <template #leading><Save :size="14" /></template>
      保存する
    </UiButton>
  </UiCard>
</AdminLayout>
```

### 5.2 データテーブル

```vue
<UiDataTable
  :columns="columns"
  :rows="items.data"
  :pagination="items"
  :row-href="(r) => route('admin.xxx.show', r.id)"
  empty-message="データがありません"
>
  <template #cell-status="{ value }">
    <UiBadge :variant="statusVariant(value)" dot>{{ value }}</UiBadge>
  </template>
  <template #cell-actions="{ row }">
    <UiButton size="sm" variant="ghost" :href="route('admin.xxx.edit', row.id)">
      <Pencil :size="13" />
    </UiButton>
  </template>
</UiDataTable>
```

### 5.3 削除確認ダイアログ

```vue
<UiDialog v-model:open="confirmOpen" title="削除">
  <p class="text-sm text-brand-text-muted">{{ target?.name }} を削除します。</p>
  <template #footer>
    <UiButton variant="ghost" @click="confirmOpen = false">キャンセル</UiButton>
    <UiButton variant="danger" :loading="deleting" @click="confirmDelete">削除する</UiButton>
  </template>
</UiDialog>
```

### 5.4 トースト

```js
import { useToast } from '@/composables/useToast.js';
const toast = useToast();
toast.success('保存しました');
toast.danger('エラー');
```
※ Laravel の `session()->with('success')` も自動で Toast に流れる（ToastContainer が shared flash を watch）。

## 7. 危険地帯（触る時の注意）

1. **`Customer/Show` / `Reservation/Show` のモーダル** — 10個以上あり、`useForm` と v-model が複雑に絡む。**フォーム送信ロジックを壊さないため**、Inputを UiInput に置き換える際は `v-model` の連携を確認すること。
2. **`Reservation/Show` のメールスレッドUI** — 展開/返信/引用ブロックの state（`expandedThreadIds`, `activeReplyThreadId` など）が複雑。UI刷新は完了済みだが、ロジックは温存。
3. **Schedule/Index.vue（FullCalendar）** — **絶対に触らない**。将来 Googleカレンダー表示に切り替えるため。
4. **公開ページ（Event/Show.vue, ReservationForm.vue）** — **絶対に触らない**。
5. **機械置換（perl）は macOS sed の `\b` が動かないので perl を使う**。`-i` の後に `''` を付けるのは macOS sed 流儀、perl は `-i` だけでよい。
