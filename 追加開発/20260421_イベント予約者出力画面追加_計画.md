# 管理画面「イベント予約者出力」画面追加 計画書（2026-04-21）

## 1. 概要

| 項目 | 内容 |

### 目的

イベント管理配下に「**イベント予約者出力**」画面を新設し、**店舗・イベント・ステータス** を縦断して予約者情報を俯瞰＋CSV一括出力できるようにする。既存の予約管理画面（イベント単位）では横断集計・出力ができず、現場での名簿作成・外部システム連携に時間がかかっている課題を解決する。

### 成果物

1. 新ルート：`admin.events.reservations.export` （または `admin.event-reservations.export` の独立リソースにするかは下部「ルート命名の選択肢」で検討）
2. 新Vueページ：`resources/js/Pages/Admin/Event/ReservationExport.vue`（仮）
3. 新Controller：`app/Http/Controllers/Admin/EventReservationExportController.php`（新規、または既存 `Admin/ReservationController` に追加）
4. サイドバーメニュー追加（「イベント管理」セクション内）

## 2. 機能仕様

### 2-1. 絞り込みエリア（4段ブロック）

画面上部に絞り込みエリアを配置。下記4つのブロックで構成し、いずれも **チェックボックス＋全選択／全解除ボタン** を持つ。

#### ブロックA：店舗フィルタ
- 全店舗を一覧表示（チェックボックス）
- チェック選択した店舗に紐づくイベントのみ、ブロックBのイベント候補に表示される（`event_shop` ピボット経由で絞り込み）
- 初期状態：全店舗チェック済み
- 操作：「全選択」「全解除」ボタン

#### ブロックB：イベントフィルタ
- ブロックAでチェックされた店舗に属するイベントのみ表示（チェックボックス）
- チェック選択したイベントに属する予約データが、下部テーブル・CSV出力対象になる
- イベント表記：`{イベント名} {(form_type ラベル)} {(開始日〜終了日)}` で明示
- 初期状態：全解除（選択しないと下部テーブルは空）
- 操作：「全選択」「全解除」ボタン
- 多数表示時はスクロール／検索ボックス

#### ブロックC：ステータスフィルタ
- 固定ラベル（5値）：`未対応` / `確認中` / `返信待ち` / `対応完了済み` / `キャンセル済み`（`EventReservation.status` の in制約に準拠）
- チェック選択したステータスの予約のみ下部テーブル・CSV出力対象
- 初期状態：`キャンセル済み` 以外の4つをチェック済み（業務的に非キャンセルを見たい場面が多いため）
- 操作：「全選択」「全解除」ボタン

#### ブロックD：UTM流入元フィルタ（追加機能）
- 対象イベントに存在する `utm_source` の値一覧を動的に生成してチェックボックス化（`NULL` は「未設定」として個別項目扱い）
- チェック選択した流入元の予約のみ下部テーブル・CSV出力対象
- 初期状態：全チェック済み
- 操作：「全選択」「全解除」ボタン
- 目的：広告費の効果検証・キャンペーン別集計を可能にする

### 2-2. 出力カラム選択エリア

- `EventReservation` の表示可能カラム 約30項目をチェックボックスで一覧
- 初期状態：主要カラム（氏名・フリガナ・電話・メール・予約日時・会場・イベント・ステータス）のみON、残りはOFF
- 「全選択」「全解除」「主要のみ」ボタン
- チェックONの列が **CSVに出力される列、かつ下部テーブルに表示される列** となる（テーブルとCSVで列を揃えることでプレビュー＝出力結果になる）

### 2-3. 予約データテーブル（下部）

- 対象行：絞り込みエリアの条件全て合致（AND）する `EventReservation` レコード
- 表示列：出力カラム選択で ON になっている列のみ
- 各行にはソート可能ヘッダ（予約日時・ステータス・会場名 等）
- フリーワード検索ボックス（氏名・フリガナ・電話・メール・紹介者名 を対象）
- ページネーション（1ページあたり50件、ユーザが50/100/200 から選択可能）
- 行クリックで予約詳細画面に遷移（新タブ）

### 2-4. CSV出力ボタン

- ヘッダ右上に **「CSV出力」** ボタン（目立つ配置）
- クリックで「現在の絞り込み条件」「選択された出力カラム」に従いCSVダウンロード
- ファイル名：`event_reservations_YYYYMMDD_HHMMSS.csv`
- 文字コード：UTF-8 BOM付（Excel 対応）
- 件数が5000件を超える場合は確認ダイアログ表示
- 選択イベントが0件の場合はボタンを非活性化＋ツールチップ「イベントを選択してください」

### 2-5. 画面上の情報表示

- 現在の絞り込みヒット件数（テーブル上部に「対象 N 件」と表示）

### 2-6. サマリ集計（追加機能）

テーブル上部に **ピル形式の件数バッジ** でサマリを表示。

#### ステータス別件数
```
[ 未対応 24 ]  [ 確認中 12 ]  [ 返信待ち 8 ]  [ 対応完了済み 78 ]  [ キャンセル済み 6 ]
```
- 絞り込み条件（店舗・イベント・UTM）に従って計算
- ステータスフィルタ自体は無視（全ステータスの内訳を常に見えるように）
- ピルをクリックするとそのステータスのみに絞り込み（ショートカット）

#### 会場別件数
```
[ 福井店(本館) 42 ]  [ 福井店(西館) 18 ]  [ 岡山店 68 ]
```
- 絞り込み条件に従って計算
- 件数が多い順に表示（先頭5会場まで、それ以上は「他 N 会場」表示で折りたたみ）

### 2-7. CSV出力履歴ログ（追加機能）

CSV出力のたびに `csv_export_logs` テーブルに1レコード記録。

記録内容：
| カラム | 内容 |
|------|------|
| user_id | 実施ユーザID |
| target | 固定値 `'event_reservations'`（将来他用途でも使用するため汎用設計） |
| filters_json | 絞り込み条件のJSON（shop_ids, event_ids, statuses, utm_sources, search） |
| columns_json | 出力カラム名の配列 |
| row_count | 実際に出力した件数 |
| file_name | 出力ファイル名 |
| created_at | 実行日時 |

画面上に履歴表示はしない（今回はログ保存のみ）。将来「CSV出力履歴一覧画面」を別途作る余地を残す。

運用上の意義：
- 個人情報の持ち出し記録（コンプライアンス対応）
- 「誰がいつ何件出力したか」の追跡可能性
- 重複出力・不正利用の検知の礎

### UI/UX設計方針

- **レイアウト自体**：既存の管理画面（`AuthenticatedLayout` + `admin.events.index` 等）を踏襲
- **絞り込み部**：グループ化＋折りたたみ可能。初期はすべて展開
- **反応性**：チェックボックス変更時はデバウンス300ms後に自動反映（再検索ボタン不要）
- **ローディング**：テーブル更新中はスケルトン表示
- **視覚的ヒエラルキー**：店舗→イベント→ステータス→出力カラム、の順で絞り込み強度が増す流れがわかるインデント・色分け
- **キーボード操作**：TABで全チェックボックスを巡回、スペースキーで切替
- **スマホ対応**：必須ではないが、絞り込み部はアコーディオン化して対応

### 5-2. 画面初期ロード（GET）

`index` メソッドで以下を Inertia props として返却：
- `shops`: 全店舗（id, name）
- `events`: 全イベント（id, title, form_type, start_date, end_date, shop_ids）— `form_type` 問わず全て対象
- `statuses`: 固定5値のラベル配列

予約データ本体はこの段階では返さない（イベント未選択なので）。

### 5-3. 予約データ取得（Inertia部分リロード）

フィルタ変更時は Inertia の `router.reload({ only: ['reservations'] })` で部分リロード。サーバサイドで絞り込みクエリを実行。

```php
public function index(Request $request) {
    $query = EventReservation::query()
        ->with(['event', 'venue', 'customer'])
        ->when($request->filled('event_ids'),
            fn($q) => $q->whereIn('event_id', $request->input('event_ids', [])))
        ->when($request->filled('statuses'),
            fn($q) => $q->whereIn('status', $request->input('statuses', [])))
        ->when($request->filled('search'), function($q) use ($request) {
            $s = $request->input('search');
            $q->where(function($sub) use ($s) {
                $sub->where('name', 'like', "%{$s}%")
                    ->orWhere('furigana', 'like', "%{$s}%")
                    ->orWhere('phone', 'like', "%{$s}%")
                    ->orWhere('email', 'like', "%{$s}%");
            });
        });

    return Inertia::render('Admin/Event/ReservationExport', [
        'shops' => Shop::orderBy('name')->get(['id', 'name']),
        'events' => Event::with('shops:id')
            ->orderBy('id', 'desc')
            ->get(['id', 'title', 'form_type', 'start_date', 'end_date']),
        'statuses' => ['未対応', '確認中', '返信待ち', '対応完了済み', 'キャンセル済み'],
        'reservations' => $request->filled('event_ids')
            ? $query->paginate($request->input('per_page', 50))->withQueryString()
            : null,
        'filters' => $request->only(['shop_ids', 'event_ids', 'statuses', 'search', 'per_page']),
    ]);
}
```

### 5-4. CSV出力（POST）

`csv` メソッドで `StreamedResponse` を返却（参考実装：`AttendanceController::exportCsv`）：

```php
public function csv(Request $request) {
    $validated = $request->validate([
        'event_ids'  => 'required|array|min:1',
        'statuses'   => 'required|array|min:1',
        'columns'    => 'required|array|min:1',  // 出力カラム名の配列
        'search'     => 'nullable|string',
    ]);

    $query = $this->buildQuery($validated);   // 5-3 と同じ条件
    $columns = $validated['columns'];
    $labels = $this->columnLabels($columns);  // ['name' => '氏名', ...]

    return new StreamedResponse(function () use ($query, $columns, $labels) {
        $stream = fopen('php://output', 'w');
        fprintf($stream, chr(0xEF) . chr(0xBB) . chr(0xBF));   // UTF-8 BOM
        fputcsv($stream, array_values($labels));
        $query->chunk(500, function ($rows) use ($stream, $columns) {
            foreach ($rows as $row) {
                fputcsv($stream, $this->rowToCsv($row, $columns));
            }
        });
        fclose($stream);
    }, 200, [
        'Content-Type' => 'text/csv; charset=UTF-8',
        'Content-Disposition' => 'attachment; filename="event_reservations_'
            . now()->format('Ymd_His') . '.csv"',
    ]);
}
```

- chunk(500) でメモリ安全
- JSON型カラムは `json_decode` して `;` 区切り文字列化
- bool は `はい / いいえ`、date/datetime は `Y-m-d` / `Y-m-d H:i` で整形
- リレーション値は `event.title`、`venue.name` 等の「ドット記法」指定で抽出

## 6. 動作フロー

```
1. 管理画面サイドバーから「イベント予約者出力」を開く
   ↓
2. 初期表示：
   - 店舗：全チェック
   - イベント：全解除（下部テーブルは空）
   - ステータス：キャンセル以外の4つON
   - 出力カラム：主要8項目のみON
   ↓
3. ユーザ操作：
   (a) 店舗チェックを変更 → イベント候補リストが絞り込まれる
   (b) イベントをチェック → 下部テーブルに予約データが表示される
   (c) ステータスチェックを変更 → テーブルがフィルタされる
   (d) 出力カラムをチェック/解除 → テーブル列が表示/非表示になる
   ↓
4. 必要な絞り込みが済んだら「CSV出力」ボタン押下
   ↓
5. サーバがCSVをStreamedResponseで返し、ブラウザが自動ダウンロード
```

## 9. 想定リスクと対策

| リスク | 対策 |
|------|------|
| 大量データでのCSVメモリ枯渇 | `chunk(500)` で逐次書き出し、`StreamedResponse` 使用 |
| N+1 クエリ | `with(['event', 'venue', 'customer'])` の eager loading 徹底 |
| フィルタ組合せ増大での性能劣化 | 主要絞り込みに `event_id` インデックスが既に存在することを前提。追加で `status` にインデックスを検討 |

| JSON型の表示が読みにくい | CSV側で `;` 区切り変換、テーブル側でも配列を可読文字列化 |
