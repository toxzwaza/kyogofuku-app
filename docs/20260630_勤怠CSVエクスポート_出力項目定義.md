# 勤怠CSVエクスポート 出力項目定義（確定版）

作成日：2026-06-30
対象：勤怠管理（`Admin/Attendance/Index`）の「CSVダウンロード」から出力する月次集計CSV
対象ブランチ：`feature/attendance-enhancement-2`

## 目的
スタッフ別・月次の勤怠集計を、下記レイアウトのCSVとしてエクスポートできるようにする。
（1行＝1スタッフ。期間内の各勤怠を集計した値を列に出力）

- 集計期間（締め）：**前月21日〜当月20日（給与締め）**。既存明細CSVと同じ。
- 日数はすべて **1日単位（半休なし）**。表記は小数2桁（例 22.00）。
- 時間は **HH:MM**（60h超も時間表記、例 188:00）。

---

## 出力対象（列順・確定）

| # | 列名 | 種別 | 定義 |
|---|------|------|------|
| 1 | 社員番号 | 識別子 | 当面は**ユーザーID（`login_id` または `users.id`）で代用** |
| 2 | 氏名 | 識別子 | `users.name` |
| 3 | 出勤日数 | 日数 | 期間内に**出勤打刻（`clock_in_at`）がある日数**（承認問わず・休日出勤を含む） |
| 4 | 休日出勤日数 | 日数 | 会社カレンダーにパターンが無い日への出勤のうち、**振替対象日が未登録**のものの日数 |
| 5 | 有給日数 | 日数 | 休暇入力機能で登録された**有給**の日数 |
| 6 | 特別休暇日数 | 日数 | 休暇入力機能で登録された**特別休暇**の日数 |
| 7 | 欠勤日数 | 日数 | 休暇入力機能で登録された**欠勤**の日数 |
| 8 | 就労時間 | 時間 | **給与用（丸め後）の在社時間 − 休憩時間** を月内合算 |
| 9 | 普通残業 | 時間 | 残業（ベース終了以降の超過分）のうち**深夜残業を除いた分** |
| 10 | 深夜残業 | 時間 | **全勤務のうち 22:00〜翌5:00 に重なる時間**（休憩控除） |
| 11 | 遅早回数 | 回数 | 遅刻・早退の回数（**1日あたり最大1回**） |
| 12 | 遅早時間 | 時間 | 遅刻分＋早退分の合計時間 |

## 出力対象外（排除）
- 代休日数 / 休日勤務時間 / 法定内残業 / 早出時間 / 法定休日勤務 / 通勤（岡山店・城東店・浜店）/ 催事時間 / 催事時間外

## 各項目の算出仕様（確定）

### 1. 社員番号
当面**ユーザーID（`login_id` または `users.id`）で代用**。正式な社員番号カラムは将来対応。

### 2. 氏名
`users.name`。

### 3. 出勤日数
期間内に**出勤打刻（`clock_in_at`）がある日数**（承認状態は問わない。休日出勤日数を含む）。半休なし＝整数。

### 4. 休日出勤日数
会社カレンダー（`company_calendar_days`）にパターン（A/B/C）が無い日への出勤について：
- **振替対象日が登録されている** → 振替勤務扱い → **休日出勤日数に含めない**。
- **振替対象日が未登録** → 純粋な休日出勤 → **休日出勤日数に含める**。
- ※ 振替対象日（代わりに休んだ出勤予定日）を選択・登録できる仕組みを別途用意する（下記「前提機能2」）。

### 5〜7. 有給日数 / 特別休暇日数 / 欠勤日数
**新規「休暇入力機能」**で登録されたデータを区分別に日数集計（区分3種・1日単位）。

### 8. 就労時間
**給与用（丸め後）の在社時間（`payroll_clock_in_at`〜`payroll_clock_out_at`）− 休憩時間** を月内合算。

### 9. 普通残業
残業（ベース終了以降の超過分。既存 `overtime_minutes_rounded`）から、**深夜帯（22:00〜翌5:00）に重なる残業分を控除**した分。

### 10. 深夜残業
**全勤務のうち 22:00〜翌5:00 に重なる時間**（残業に限らない）。深夜帯にかかる休憩は控除。

### 11. 遅早回数
ベース時刻（`resolveBaseWindow`）に対し、出勤がベース開始より遅い＝遅刻／退勤がベース終了より早い＝早退。
- **1分でも遅刻・早退とする**（猶予なし）。
- **1日あたり最大1回**（遅刻と早退が両方でも1回）。
- **パターン上書きで解決した日（ベース時刻が出せる日）のみ**判定対象。ベース未解決日はカウント対象外。

### 12. 遅早時間
遅刻分（出勤−ベース開始）＋早退分（ベース終了−退勤）の合計。前提（猶予なし・ベース解決日のみ）は遅早回数と同じ。

## 本CSVの前提となる新規実装機能

このCSVを実現するには、以下の機能が前提として必要（CSV単体では完結しない）。

1. **休暇入力機能**（有給/特別休暇/欠勤）
   - 区分3種、**1日単位（半休なし）**。
   - UI：勤怠管理画面に**「休暇登録」ボタン**を設け、押下で**専用モーダル**を開いて登録。
   - データ：新規テーブル（例 `attendance_leaves`: `user_id`, `date`, `leave_type`[有給/特別休暇/欠勤]）。

2. **日次パターン上書き＋振替対象日登録**
   - 既に別途計画中の「日次パターン上書き（`pattern_override`）」機能に、**振替対象日の選択・登録**を追加。
   - 休日出勤日数（#4）の判定と、遅早判定（#11/#12：上書き解決日のみ対象）に使用。

3. **月次集計CSVエクスポート**
   - 既存 `exportCsv()`（`AttendanceController` L196-301）は**1打刻=1行**の明細。
   - 本CSVは**1スタッフ=1行の集計**のため、新しい出力（別エンドポイント or 出力モード切替）を実装。
   - 集計期間は前月21日〜当月20日（既存の期間フィルタを流用）。

## 確定した決定事項（Q&A 履歴）
- Q0 社員番号 → 当面ユーザーIDで代用。
- Q1-a 出勤日数 → 出勤打刻があれば1日（承認問わず）。
- Q1-b 半休 → なし（1日単位）。
- Q2 休日出勤の振替 → 振替対象日 登録済=含めない／未登録=含める。
- Q3 休暇データ → 新規「休暇入力機能」。区分3種・1日単位・勤怠管理画面の休暇登録ボタン→モーダル。
- Q4 就労時間 → 給与用（丸め後）在社時間 − 休憩時間。
- Q5 普通残業 → 残業から深夜残業を除いた分。
- Q6 深夜残業 → 全勤務のうち深夜帯（22:00〜翌5:00）に重なる時間。
- Q7-a 遅早回数 → 1日あたり最大1回。
- Q7-b 遅早しきい値 → 1分でも遅刻・早退（猶予なし）。
- Q7-c ベース未解決日 → パターン上書きで解決した日のみ対象。
- Q8 集計期間 → 前月21日〜当月20日（給与締め）。
