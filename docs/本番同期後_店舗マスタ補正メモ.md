# 本番同期後 店舗マスタ補正メモ

## 目的

本番DBのダンプをローカル（`localdb`）に取り込んだ際、`shops` テーブルの **Googleカレンダー連携** と **LINE Group ID** はローカル環境用の値で上書きする必要がある。
本番と同じ値のままだと、ローカルでのテスト操作が本番カレンダー / 本番LINEグループへ通知・反映されてしまう。

本番DB → ローカル取り込みを実施した直後に、以下のSQLを必ず適用すること。

## 補正用SQL（本番→ローカル取り込み直後に実行）

```sql
-- 岡山店・城東店: Googleカレンダー & LINEグループをローカル検証用に差し替え
UPDATE shops
SET
  google_calendar_id = 'fd488d9a46c6581a5653a642b67064cd491f2d4eeaf17aa675a625e62df99882@group.calendar.google.com',
  line_group_id      = 'C4bbc8f222d645a6aa65506598b29b020'
WHERE id IN (1, 2);

-- 浜店・福井店・テスト店: Googleカレンダーは未連携、LINEは検証用グループのみ
UPDATE shops
SET
  google_calendar_id = NULL,
  line_group_id      = 'C4bbc8f222d645a6aa65506598b29b020'
WHERE id IN (3, 4, 5);

-- 補正後の確認
SELECT id, name, google_calendar_id, line_group_id FROM shops ORDER BY id;
```

## 運用ルール

1. 本番DBダンプをローカルに取り込んだら、**最初に上記SQLを必ず実行する**。
2. 新店舗追加や Googleカレンダー連携店舗が増えた場合は、本ファイルを更新する。
3. 本番側の `line_group_id` / `google_calendar_id` を直接ローカルから書き換えないこと。
