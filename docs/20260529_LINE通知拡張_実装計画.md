# LINE連携 通知拡張 実装計画（2026-05-29）

別エージェントが本ドキュメントだけを読んで実装に着手できるよう、現状・要件・変更内容・テスト方針を完結にまとめる。

---

## 1. 背景・要件

### 1.1 ユーザー要望（原文）

> LINE連携時の機能を拡張したい。追加する機能として、
> 1. 友達登録されたタイミングで「氏名さんが公式LINE登録を完了しました。」というメッセージを通知
> 2. システム連携したタイミングで「氏名さんがシステム連携が完了しました。」のメッセージを送信
> 3. 連携したユーザーが公式LINEへメッセージを送信したタイミングで「氏名さんからメッセージを受信しました。」に改行して、実際のメッセージを表示する

### 1.2 設計上の確定事項（要件すり合わせ済み）

| # | 論点 | 確定方針 |
|---|------|---------|
| A | 通知先 | **店舗LINEグループ**（`shops.line_group_id`）。`CustomerLineContact.shop_id` から店舗を特定する。 |
| B | 友達追加（follow）時点の通知 | **スコープ外**。友達追加だけでは紐付け・店舗・氏名が確定しないため、本回は実装しない。 |
| C | メッセージ受信通知のリンクボタン | **付ける**。Flex メッセージ＋URIボタンで admin の顧客詳細／予約詳細ページへ遷移可能にする。 |

### 1.3 今回の実装対象（最終スコープ）

1. **システム連携完了時**：店舗LINEグループへ「{氏名}さんがシステム連携が完了しました。」を通知
2. **連携済みユーザーからのメッセージ受信時**：店舗LINEグループへ「{氏名}さんからメッセージを受信しました。\n{本文}」を Flex + URIボタンで通知

## 3. 実装方針

### 3.1 ハイレベル設計

店舗グループ通知のロジックを集約した新規サービス `ShopLineGroupNotifier` を1本切り出し、以下3箇所から呼び出す：

```
┌───────────────────────────────────────┐
│ LineMessagingWebhookController         │
│  ├ INBOUND メッセージ保存後             │──┐
│                                         │  │
│ LineLiffController                      │  │
│  ├ completeForCustomer()                │──┤   ShopLineGroupNotifier
│  └ completeForReservation()             │──┼─► ├ notifySystemLinked()
│                                         │  │   └ notifyInboundMessage()
│ LineWelcomeLinkController               │  │           │
│  ├ linkToCustomer()                     │──┤           ▼
│  └ linkToReservation()                  │──┘   LineWebhookController
└───────────────────────────────────────┘       ::pushToLineGroup() を再利用
```

### 3.2 設計原則

- **既存のユーザー本人へのあいさつ Push は一切変更しない**（`sendLinkWelcomeToLineUser` / `sendWelcomePushAndRecord` はそのまま）
- 通知失敗は **`Log::warning` のみで握りつぶす**（既存 `BladeLp/LineNotifier` と同じ方針、業務処理を止めない）
- 既存の `pushToLineGroup` を再利用するため、Flex メッセージや HTTP クライアントを新規に書かない
- 二重通知防止：メッセージ受信は既存の `line_message_id` 重複ガード（L87-92）の **後段** に差し込む

## 4. 変更詳細（ファイル単位）

### 4.1 新規：`app/Services/Line/ShopLineGroupNotifier.php`

```php
<?php

namespace App\Services\Line;

use App\Http\Controllers\LineWebhookController;
use App\Models\CustomerLineContact;
use Illuminate\Support\Facades\Log;

class ShopLineGroupNotifier
{
    /**
     * システム連携完了を店舗グループに通知する。
     */
    public function notifySystemLinked(CustomerLineContact $contact): void
    {
        $groupId = $this->resolveGroupId($contact);
        if ($groupId === null) {
            return;
        }

        $name = $this->resolveName($contact);
        $message = "{$name}さんがシステム連携が完了しました。";
        $detailUrl = $this->resolveDetailUrl($contact);

        $this->push($groupId, $message, $detailUrl, $contact, 'system_linked');
    }

    /**
     * 連携済みユーザーから受信したメッセージを店舗グループに通知する。
     */
    public function notifyInboundMessage(CustomerLineContact $contact, string $text): void
    {
        $groupId = $this->resolveGroupId($contact);
        if ($groupId === null) {
            return;
        }

        $name = $this->resolveName($contact);
        $message = "{$name}さんからメッセージを受信しました。\n{$text}";
        $detailUrl = $this->resolveDetailUrl($contact);

        $this->push($groupId, $message, $detailUrl, $contact, 'inbound_message');
    }

    /**
     * 紐付け先の氏名を解決する。
     * 優先順位: customer.name → event_reservation.name → contact.label → 'お客様'
     */
    private function resolveName(CustomerLineContact $contact): string
    {
        $contact->loadMissing(['customer', 'eventReservation']);

        if ($contact->customer && filled($contact->customer->name)) {
            return (string) $contact->customer->name;
        }
        if ($contact->eventReservation && filled($contact->eventReservation->name)) {
            return (string) $contact->eventReservation->name;
        }
        if (filled($contact->label)) {
            return (string) $contact->label;
        }
        return 'お客様';
    }

    /**
     * 通知先の店舗グループ ID を解決する。
     */
    private function resolveGroupId(CustomerLineContact $contact): ?string
    {
        $contact->loadMissing('shop');
        $gid = $contact->shop?->line_group_id;
        return filled($gid) ? (string) $gid : null;
    }

    /**
     * 詳細ページの URL を解決する。
     * customer 優先、なければ event_reservation。
     */
    private function resolveDetailUrl(CustomerLineContact $contact): ?string
    {
        if ($contact->customer_id) {
            return route('admin.customers.show', $contact->customer_id, true);
        }
        if ($contact->event_reservation_id) {
            return route('admin.reservations.show', $contact->event_reservation_id, true);
        }
        return null;
    }

    private function push(string $groupId, string $message, ?string $detailUrl, CustomerLineContact $contact, string $kind): void
    {
        try {
            $line = app(LineWebhookController::class);
            if ($detailUrl !== null) {
                $line->pushToLineGroup($message, $groupId, $detailUrl, '詳細を開く');
            } else {
                $line->pushToLineGroup($message, $groupId);
            }
        } catch (\Throwable $e) {
            Log::warning('[ShopLineGroupNotifier] LINE通知失敗', [
                'kind' => $kind,
                'contact_id' => $contact->id,
                'shop_id' => $contact->shop_id,
                'group_id' => $groupId,
                'error' => $e->getMessage(),
            ]);
        }
    }
}
```

#### 実装メモ
- `LineWebhookController` は `app()` 経由でインスタンス化する（既存 `BladeLp/LineNotifier` と同様の慣例）
- ボタンラベルは `'詳細を開く'`（既存 `pushToLineGroup` のデフォルト挙動と一致させる）
- ルート名の最終確認：`admin.customers.show` / `admin.reservations.show` が存在することは `routes/web.php` で要確認（実装エージェントは grep で実在確認すること）

#### 4.3.4 新規 private helper

```php
private function wasNewlyLinkedForCustomer(?CustomerLineContact $existing, CustomerLineLinkToken $link): bool
{
    if ($existing === null) {
        return true; // 完全新規
    }
    return (int) $existing->customer_id !== (int) $link->customer_id;
}

private function wasNewlyLinkedForReservation(?CustomerLineContact $existing, CustomerLineLinkToken $link): bool
{
    if ($existing === null) {
        return true;
    }
    return (int) $existing->event_reservation_id !== (int) $link->event_reservation_id;
}
```

#### 重要：判定タイミング
`updateOrCreate` 実行 **前** の `$existing` の状態で判定する必要があるため、`DB::transaction` 呼び出し前に値を保持しておく。`$existing` は既存コード（L155, L232 付近）で取得済みなのでそのまま使える。

## 6. スコープ外（今回は実装しない）

- **友達追加（follow）時の通知**：紐付け前のため店舗・氏名を確定できない。要件討議で除外決定。
- **unfollow（友だち解除）時の通知**：今回の要望に含まれない。
- **画像・スタンプ等の非テキストメッセージ通知**：現状通り `LineUnknownInboundMessage` 行き。本文表示ができないため通知対象外。
- **管理画面の通知一覧 UI**：店舗グループ LINE に飛ぶため不要。
- **`sendLinkWelcomeToLineUser` / `sendWelcomePushAndRecord` の挙動変更**：ユーザー本人への既存 Push はそのまま温存。

## 7. 受け入れ条件（Acceptance Criteria）

1. ✅ 連携済みユーザーが公式LINEへテキストメッセージを送ると、該当店舗の LINE グループに「{氏名}さんからメッセージを受信しました。\n{本文}」が Flex + URIボタン付きで届く
2. ✅ URIボタンを押すと admin の顧客詳細（or 予約詳細）ページが開く
3. ✅ link token 経由の連携・あいさつメッセージ経由の自己紐付け、どちらでも連携成立時に店舗グループに「{氏名}さんがシステム連携が完了しました。」が届く
4. ✅ 既存のユーザー本人へのあいさつ Push（`link_welcome_text`）は引き続き動作する
5. ✅ 友達追加（follow）だけでは何も通知されない
6. ✅ `shop.line_group_id` が空の店舗では通知がスキップされ、例外でユーザー操作が阻害されない
7. ✅ LINE API 呼び出しが失敗しても、ユーザー側の連携・メッセージ受信処理は成功扱いで完了する
8. ✅ 同一 `line_message_id` の Webhook 再送で重複通知が発生しない
9. ✅ 全テストが通る（`php artisan test`）

## 8. 留意事項・実装時の確認ポイント

### 8.1 ルート名の実在確認

`route('admin.customers.show', ...)` / `route('admin.reservations.show', ...)` が `routes/web.php` に存在することを実装前に grep で確認すること。命名が異なる場合は実在する名前に置き換える。

```bash
grep -nE "(customers\.show|reservations\.show)" routes/web.php
```

### 8.2 `LineWebhookController` のインスタンス化

既存 `BladeLp/LineNotifier` は `new LineWebhookController` を直接使っているが、新サービスでは `app(LineWebhookController::class)` でコンテナ経由にする。これによりテストでモック差し替えが容易になる。

### 8.3 Flex メッセージの本文長制限

LINE Flex メッセージのテキストは長文だと省略される。メッセージ受信通知では本文がそのまま入るため、極端に長いメッセージ（例：2000文字超）は注意。現状の `pushToLineGroup` は `mb_substr` 制限を本文側にかけていないので、必要に応じて `mb_substr($text, 0, 1500)` のような上限を `ShopLineGroupNotifier::notifyInboundMessage` 内で設けるか検討する（PR時に判断）。

### 8.4 `EventReservation::name` の存在確認

`EventReservation` モデルに `name` 属性があることは `BladeLp/LineNotifier::buildMessage()` 内で `{$reservation->name}` が使われていることから確認済み。

### 8.5 ログのキー設計

`Log::warning` で出すキーは以下を含めると運用時に追跡しやすい：
- `kind` (system_linked / inbound_message)
