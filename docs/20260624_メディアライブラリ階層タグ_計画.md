# メディアライブラリ 階層タグ機能 実装計画

作成日: 2026-06-24 / 担当: システム開発部門 / ブランチ: `feature/media-library`

## 背景・目的
メディアライブラリのタグを階層構造（フォルダ風）にして管理しやすくする。
例: `2026 > イベント > イベント1`

要件:
1. タグを階層構造で設定できる（親子関係）
2. メディアには複数タグを付与（従来どおり）。付与は**セレクトボックス**で追加登録
3. タグで絞り込む際、**指定タグの子要素（子孫）も検索結果に含める**

## 現状
- タグは `media_files.tags`（JSON配列カラム）に自由入力文字列で保存。階層なし・正規化なし
- 付与: カンマ区切りテキスト入力（アップロード時・詳細編集時）
- 絞り込み: 単一セレクト → `whereJsonContains('tags', $tag)`（完全一致のみ）
- 利用箇所: `MediaFileController`（index/store/storeJson/listJson/update）、`Media/Index.vue`＋`IndexLegacy.vue`、`MediaPicker.vue`、`LineMessageMediaStore.php`（LINEメディアを `line_inbound`/`line_outbound` で自動タグ付け）
- 参考: 顧客タグ（`CustomerTag` + `customer_tag_assignments` 多対多）が多対多の手本（ただしフラット）

## 決定事項
- タグ管理UI: **ライブラリ内モーダル**
- 付与UI: **パス付きセレクト＋チップ**（1つずつ追加、×で解除）
- 対応範囲: **新UI（Index.vue）のみ**（Legacyは現状維持）

## データモデル
```
media_tags                          media_file_tag（ピボット）
├ id                                ├ media_file_id
├ parent_id (nullable・自己参照)    ├ media_tag_id
├ name                              └ timestamps
├ sort_order                        unique(media_file_id, media_tag_id)
└ timestamps
   unique(parent_id, name)
```
- 階層 = `parent_id` 自己参照ツリー
- メディア⇔タグ = 多対多
- 子孫含む絞り込み = ツリーをロード → 選択タグ＋全子孫IDを算出 → `whereHas('tags', whereIn(media_tag_id, [...]))`（小規模ツリー前提でPHP側計算、DB非依存）

## 実装フェーズ

### Phase 1 — バックエンド基盤
1. マイグレーション: `media_tags`・`media_file_tag` 作成
2. データ移行マイグレーション: 既存タグ文字列（`line_inbound`/`line_outbound` 含む全distinct値）をフラットタグ化し紐付け。旧 `tags` カラムは保持
3. `MediaTag` モデル（parent/children/mediaFiles/`descendantIds()`/`fullPath`）、`MediaFile::tags()` belongsToMany 追加
4. `MediaTagController`（index/store/update/destroy/move）＋ルート

### Phase 2 — メディア側ロジック
5. `MediaFileController`: `tag_ids[]` 受付＆ピボット同期（store/storeJson/update）、絞り込みを子孫含むに変更、サイドバー用にタグツリー返却
6. `LineMessageMediaStore`: LINEメディアをリレーションで自動タグ付け（無ければ作成）

### Phase 3 — フロント（新UIのみ）
7. `Media/Index.vue`: 付与UI（パス付きセレクト＋チップ）、サイドバーのツリー絞り込み（子孫含む）、タグ管理モーダル（作成・リネーム・移動・削除・並び替え）
8. `MediaPicker.vue`: ツリー絞り込み・子孫対応

### Phase 4 — 検証
9. 単体テスト（子孫絞り込み・移行・付与同期）、フロントビルド、任意でPlaywright目視

## 注意点・残課題
- 旧 `tags` カラムは確認後の別マイグレーションで削除（即時削除しない）
- 本番反映時は `php artisan migrate` 必須
- 循環参照防止: タグ移動（親付け替え）時に自分自身・子孫を親に指定できないようバリデーション
- タグ削除時の挙動: 子タグの扱い（連鎖削除 or 親へ繰り上げ）と、紐付くメディアのピボット解除を定義
