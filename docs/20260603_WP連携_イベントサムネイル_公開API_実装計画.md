# WP連携：イベントサムネイル機能 ＋ 公開API 実装計画

- **作成日**：2026-06-03
- **対象システム**：kyogofuku-app（Laravel）
- **連携先**：
  - `kyogofuku-kouichilocal`（WordPressテーマ `kouichi`）→ **岡山店 / 城東店 / 浜店** のイベント（重複イベントは1件に集約）
  - `kyogufuku-hiratalocal`（WordPressテーマ `hirata`）→ **福井店** のイベント
- **目的**：WordPress 2サイトの「Pick Up」セクションと footer 固定バナーを、kyogofuku-app の公開イベントから API 経由で取得・表示する

---

## 1. 背景・現状

### システム側（kyogofuku-app）
- `events` テーブル：`slug`, `title`, `description`, `start_at`, `end_at`, `is_public`, `form_type` 等を保持。**サムネイル専用カラムは無し**
- `event_images` テーブル：LP表示用の複数画像（順序付き）を保持。`MediaFile` と連携、S3対応済み
- 店舗は `shops` テーブル＋`event_shop` 中間テーブルで多対多。店舗マスタは「岡山店 / 城東店 / 浜店 / 福井店」
- LP公開URLは `https://kyogofuku-event.com/event/{slug}` 形式
- `routes/api.php` は X-Api-Key 認証の社内API中心。**公開API エンドポイントはまだ無し**

### WordPress側
- kouichi / hirata それぞれ独自テーマ
- 現状 Pick Up は WordPress の `event` カテゴリ投稿を `WP_Query` で 10件取得（`index.php`）
- footer 固定バナー（`l-fixedBanner`）は **画像URLとリンクがハードコード**（1件のみ、`footer.php`）

実装イメージ：

```php
public function getThumbnailUrlAttribute(): ?string
{
    $path = $this->thumbnail_path;
    if (!$path) return null;
    if (str_starts_with($path, 'http')) return $path;
    if (($this->thumbnail_storage_disk ?? 'public') === 's3') {
        $path = str_replace('\\', '/', $path);
        return Storage::disk('s3_public')->url($path);
    }
    return asset('storage/' . $path);
}

public function thumbnailMediaFile()
{
    return $this->belongsTo(MediaFile::class, 'thumbnail_media_file_id');
}
```

## 5. Phase 2：公開API作成

### 5-1. ルート定義

`routes/api.php` に追加：

```php
use App\Http\Controllers\Api\PublicEventController;

Route::prefix('public')->group(function () {
    Route::get('/events', [PublicEventController::class, 'index']);
    Route::get('/events/footer-banner', [PublicEventController::class, 'footerBanner']);
});
```

### 5-2. コントローラ

**新規ファイル**：`app/Http/Controllers/Api/PublicEventController.php`

#### `GET /api/public/events`

**クエリパラメータ**：
| パラメータ | 必須 | 既定 | 備考 |
|-----------|------|-----|------|
| `shop` | ○ | — | 店舗名。**カンマ区切りで複数指定可**。例：`岡山店,城東店,浜店` または `福井店` |
| `limit` | × | 10 | 最大 20 |

**条件**：
- `is_public = true`
- **`form_type IN ('reservation', 'reservation_hakama')`**（振袖予約 / 袴予約のみ）
- `shops.name IN ({shop})` のいずれかを含むイベント
- **`end_at IS NULL` または `end_at >= 今日`**（過去終了は除外）
- **重複排除**：1イベントが複数店舗に紐づいていても、結果は **1件のみ**（`SELECT DISTINCT events.*` または `groupBy('events.id')`）

**並び順**：
- `end_at` 昇順、NULL は末尾
  ```sql
  ORDER BY (end_at IS NULL) ASC, end_at ASC, id ASC
  ```

**件数**：`limit` 分

#### `GET /api/public/events/footer-banner`

**クエリパラメータ**：
| パラメータ | 必須 | 備考 |
|-----------|------|------|
| `shop` | ○ | 店舗名。**カンマ区切りで複数指定可**。例：`岡山店,城東店,浜店` または `福井店` |

**条件**：
- `is_public = true`
- **`form_type IN ('reservation', 'reservation_hakama')`**（振袖予約 / 袴予約のみ）
- `shops.name IN ({shop})` のいずれかを含む
- `end_at >= 今日`（未入力は **除外**）
- **重複排除**：複数店舗に紐づくイベントも 1件として扱う

**並び**：`end_at` 昇順、先頭 1件

**該当なし時**：HTTP 204 No Content を返す

### 5-3. レスポンススキーマ（共通）

```json
{
  "data": [
    {
      "id": 1,
      "slug": "souden_2026_05_okayama",
      "title": "創業祭",
      "description": "...",
      "shop_names": ["岡山店", "城東店"],
      "start_at": "2026-06-01",
      "end_at": "2026-06-15",
      "thumbnail_url": "https://.../thumbnail.webp",
      "first_image_url": "https://.../event_image_1.webp",
      "lp_url": "https://kyogofuku-event.com/event/souden_2026_05_okayama"
    }
  ]
}
```

| フィールド | 内容 | 使用場面 |
|-----------|------|---------|
| `thumbnail_url` | `events.thumbnail_path` から生成（Phase 1 で追加するサムネイル画像） | **Pick Up セクション**で使用 |
| `first_image_url` | `event_images` のうち `sort_order` 最小のレコードの `url`（`webp_path` があれば優先） | **footer バナー**で使用 |
| `lp_url` | `https://kyogofuku-event.com/event/{slug}` | Pick Up / footer バナーのクリック先 |

- `lp_url` のベースURLは `config/app.php` に `event_lp_base_url`（既定 `https://kyogofuku-event.com`）を追加して構築
- `shop_names` は **配列**で、そのイベントに紐づく全店舗名を返す（複数店舗対応のため `shop_name` 単数ではなく配列に変更）
- `thumbnail_url` / `first_image_url` は登録がなければ `null` を返す
- 実装上はコントローラで `Event` モデルに `->with('images')` を eager load し、`images->first()` から `first_image_url` を組み立てる（リレーション側で既に `sort_order` 昇順）

### 5-5. キャッシュ

- コントローラ内で `Cache::remember('public_events:shops={shop1,shop2,...}:limit={limit}', 300, fn() => ...)`
  - キャッシュキー生成時は `shop` パラメータをソートして正規化（`岡山店,城東店,浜店` と `浜店,岡山店,城東店` を同一キーに）
- footer-banner も同様に 300秒キャッシュ
- **キャッシュ無効化**：
  - `EventObserver` を新設して `saved` / `deleted` 時に該当店舗のキャッシュをクリア
  - シンプルにキャッシュタグ or プレフィックス一括削除（`public_events:*`）で運用

### 5-6. テスト

`tests/Feature/Api/PublicEventControllerTest.php`（新規）

検証項目：
- 単一店舗フィルタが効くこと（`shop=福井店` → 岡山店イベント混入なし）
- **複数店舗フィルタが効くこと**（`shop=岡山店,城東店,浜店` → 福井店のみのイベントは混入なし）
- **重複排除が効くこと**（岡山店＋城東店の両方に紐づくイベントが 1件のみ返ること）
- `shop_names` 配列が紐づく全店舗を含むこと
- `is_public = false` が含まれないこと
- `end_at` 昇順、NULL は末尾
- `limit` パラメータが効くこと（既定10、最大20）
- footer-banner：`end_at >= 今日` の最も近い1件を返すこと
- footer-banner：複数店舗指定でも重複なく1件返ること
- footer-banner：該当なしで 204 を返すこと
- レスポンスに `thumbnail_url` / `first_image_url` / `lp_url` が正しく含まれること
- `first_image_url` が `event_images` の `sort_order` 最小レコードを参照していること（複数画像登録時の検証）
- `event_images` 未登録時に `first_image_url` が `null` となること
