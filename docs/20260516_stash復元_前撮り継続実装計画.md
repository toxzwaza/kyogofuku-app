## 5. Phase 2: stash から「前撮り中核改修」だけを抽出反映

stash は単純な apply ではなく、**ファイル指定で部分 checkout** する。

### 5-1. 新規ファイル4つを stash から取り出し
```bash
git checkout stash@{0} -- app/Console/Commands/ImportPreshootCommand.php
git checkout stash@{0} -- app/Console/Commands/SyncPreshootCalendarCommand.php
git checkout stash@{0} -- app/Observers/PhotoSlotObserver.php
```

> 新規ファイルは stash の untracked-parent (`stash@{0}^3`) に居る点に注意。
> `git checkout stash@{0}^3 -- <path>` が必要なら切替える。

### 5-2. 既存ファイルの変更4つを取り出し
```bash
git checkout stash@{0} -- app/Http/Controllers/Admin/PhotoSlotController.php
git checkout stash@{0} -- app/Services/GoogleCalendarSyncService.php
git checkout stash@{0} -- app/Providers/AppServiceProvider.php
git checkout stash@{0} -- routes/web.php
git checkout stash@{0} -- resources/js/Pages/Admin/PhotoSlot/Index.vue
```

### 5-3. main最新との整合性確認

```bash
# 各ファイルが main からの分岐 (8315fe4) ベースで上書きされた状態
# main側で同ファイルがその後変更されていないかチェック
for f in \
  app/Http/Controllers/Admin/PhotoSlotController.php \
  app/Services/GoogleCalendarSyncService.php \
  app/Providers/AppServiceProvider.php \
  routes/web.php \
  resources/js/Pages/Admin/PhotoSlot/Index.vue ; do
    echo "=== $f ==="
    git log --oneline 8315fe4..main -- "$f"
done
```

→ main側で更新があった場合は **手動マージ**（stash の意図を活かしつつ main の修正も取り込む）。

## 8. Phase 5: 継続実装（CSV取込みの実行）

ここからが「続き」。`20260427_前撮りデータ統合計画.md` の Phase 1〜6 を **localdb_prod 上で先に試験** する。

### 8-1. dry-run 実行
```bash
php artisan preshoot:import \
  "/Users/to-murakami/業務データ/CC_会社/システム開発部門/京呉服平田/顧客データ移行/前撮りリスト.csv" \
  --dry-run
```
→ コマンドが `--dry-run` をサポートしているか要確認。無ければ Phase 5-A で実装追加。

### 8-2. 既存 photo_slots 退避＆クリア（localdb_prod上）
```bash
# 退避CSV
docker exec akioka-mysql mysqldump -uto_murakami -pMurakami0819 \
  --no-tablespaces localdb_prod photo_slots photo_slot_shop staff_schedules \
  > "$APP_DIR/../dump/localdb_prod_photo_pre_import_$(date +%Y%m%d).sql"

# StaffSchedule 経由でGoogleカレンダーから削除（Service呼び出し）
php artisan tinker --execute="
  \App\Models\StaffSchedule::whereNotNull('photo_slot_id')->each(function (\$s) {
    app(\App\Services\GoogleCalendarSyncService::class)->deleteFromShopCalendars(\$s);
    \$s->delete();
  });
"

# photo_slots 全削除（CASCADE で photo_slot_shop も連動削除）
docker exec -i akioka-mysql mysql -uto_murakami -pMurakami0819 localdb_prod -e "
  SET FOREIGN_KEY_CHECKS=0;
  DELETE FROM photo_slots;
  ALTER TABLE photo_slots AUTO_INCREMENT=1;
  SET FOREIGN_KEY_CHECKS=1;
"
```

### 8-3. CSV インポート本番実行
```bash
php artisan preshoot:import \
  "/Users/to-murakami/業務データ/CC_会社/システム開発部門/京呉服平田/顧客データ移行/前撮りリスト.csv"
```
