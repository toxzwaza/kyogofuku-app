# 勤怠機能拡張 実装計画（最終版）

作成日：2026-06-30
対象ブランチ：`feature/attendance-enhancement-2`（`main` から分岐）
関連仕様：`docs/20260630_勤怠CSVエクスポート_出力項目定義.md`（確定版）

## Context（背景・目的）

最終ゴールは、勤怠管理（`Admin/Attendance/Index`）の「CSVダウンロード」から、スタッフ別・月次の
集計CSV（12列）を出力できるようにすること。ヒアリングの結果、このCSVは単体で完結せず、
**3つの連動機能**が必要と確定した。依存関係順に実装する。

```
①日次パターン上書き＋振替対象日   →  ②休暇入力機能（有給/特別休暇/欠勤）  →  ③月次集計CSV
（ベース時刻・休日出勤・遅早の土台）     （休暇日数の入力元）                  （①②を集計）
```

共通前提（確定仕様）：
- 集計期間＝**前月21日〜当月20日（給与締め）**、日数は**1日単位（半休なし）**・小数2桁表記、時間は **HH:MM**（60h超も時間表記）。

---

## ① 日次パターン上書き ＋ 振替対象日登録

### 目的
会社カレンダー（A/B/C・全社一律）に無い日に振替出勤した日でも、スタッフ個別にパターンを指定して
**ベース出勤・退勤時刻を算出**できるようにする。あわせて**振替対象日**（代わりに休んだ出勤予定日）を登録し、
休日出勤日数の判定に使う。パターン未設定日に出勤している行は管理画面で**オレンジ強調**し、詳細モーダルで設定する。

### 変更（バックエンド）
1. マイグレーション新規：`attendance_records` に追加
   - `pattern_override` string(1) nullable（A/B/C）
   - `substitute_for_date` date nullable（振替対象日。登録ありなら振替勤務＝休日出勤に含めない）
2. `app/Models/AttendanceRecord.php`：`$fillable` に2列追加、`substitute_for_date` を `date` キャスト。
3. `app/Services/AttendancePayrollTimeService.php`（中核）
   - `resolveBaseWindow(User, Carbon, ?array $patternsByDate, ?string $overridePattern = null)`：
     `$pattern = ($overridePattern ?: $this->calendarPatternForDate(...))` に変更。
   - `payrollPayloadForRecord()`：`resolveBaseWindow(..., $record->pattern_override)` を渡す。
     ペイロードに追加：`applied_pattern`（上書き優先で実適用したパターン）、
     `needs_pattern`（`clock_in_at !== null && base_start_at === null`＝打刻あり・ベース未解決＝オレンジ判定）。
4. `app/Http/Controllers/Admin/AttendanceController.php::update()`
   - バリデーション追加：`pattern_override` => `nullable|in:A,B,C`、`substitute_for_date` => `nullable|date`。
   - `$record->update([... 'pattern_override' => ..., 'substitute_for_date' => ...])`（version++ は既存どおり）。
   - ※ `index()` は `$r->toArray()` で2列が自動でprops化される。

### 変更（管理画面 `resources/js/Pages/Admin/Attendance/Index.vue`）
5. 行のオレンジ強調：テーブル表示・ユーザー毎表示の各 `<tr>` に `payroll.needs_pattern` のとき `bg-orange-50` ＋
   「要パターン設定」バッジ。既存のセル単位色分け（`payrollClockInClass`）は維持。
6. 詳細モーダル（L213-273）に追加：
   - 「適用パターン」`<select v-model="editForm.pattern_override">`（A/B/C／未設定）。
   - 「振替対象日」`<input type="date" v-model="editForm.substitute_for_date">`（任意。会社カレンダー出勤日のうち休んだ日）。
   - `openEditModal`/`submitEdit` に2項目を追加（既存の連携パターンに倣う）。

## ② 休暇入力機能（有給／特別休暇／欠勤）

### 目的
CSVの有給日数・特別休暇日数・欠勤日数の入力元。区分3種・1日単位。
勤怠管理画面の「休暇登録」ボタン→専用モーダルで登録する。

### 変更（バックエンド）
1. マイグレーション新規：`attendance_leaves` テーブル
   - `id`, `user_id`(FK), `date`(date), `leave_type` string（`paid`=有給 / `special`=特別休暇 / `absence`=欠勤）,
     `note` nullable, `created_by`(FK users) nullable, timestamps。
   - `unique(['user_id','date'])`（1スタッフ1日1区分）。
2. `app/Models/AttendanceLeave.php`：新規モデル（`user()` belongsTo、定数 `TYPE_PAID/TYPE_SPECIAL/TYPE_ABSENCE`）。
3. `app/Http/Controllers/Admin/AttendanceLeaveController.php`：新規。`store()`（updateOrCreate）/`destroy()`。
   - 権限は `AttendanceScopeService::canAccessManagement` ＋ 対象ユーザーのスコープ確認（既存に倣う）。
4. ルート（`routes/web.php` 勤怠ブロック）：
   - `POST /attendance/leaves` → `admin.attendance.leaves.store`
   - `DELETE /attendance/leaves/{leave}` → `admin.attendance.leaves.destroy`
5. `Admin/AttendanceController::index()`：期間内の `attendance_leaves` も取得し、props に渡す（行表示・モーダル初期値用）。

### 変更（管理画面 Index.vue）
6. ツールバーに「休暇登録」ボタン → 専用モーダル：対象スタッフ・日付・区分（有給/特別休暇/欠勤）を選んで登録。
   既存スタッフ（`users`）・期間から候補を出す。登録済み休暇は一覧/該当行に表示し、削除可能に。

## ③ 月次集計CSVエクスポート（1スタッフ＝1行）

### 目的
①②を集計し、確定12列のCSVを出力。既存 `exportCsv()`（1打刻=1行の明細）とは別物のため新規。

### 変更（バックエンド）
1. `app/Services/AttendancePayrollTimeService.php` に集計用ヘルパを追加：
   - `nightWorkMinutes($record, $baseStart, $baseEnd, $setting)`：勤務時間（休憩除く）のうち 22:00〜翌5:00 重複分。
   - `normalOvertimeMinutes(...)`：`overtime_minutes_rounded` − 残業帯の深夜重複分（普通残業）。
   - 遅刻分・早退分（ベース解決日のみ・猶予なし・1日最大1回）。
2. `app/Http/Controllers/Admin/AttendanceController.php`：`exportSummaryCsv(Request)` を新規。
   - 期間（既定 前月21〜当月20、`from`/`to` で上書き）でユーザー単位にループし、各列を集計。
   - 列定義（確定）：
     | 列 | 集計 |
     |----|------|
     | 社員番号 | `login_id`（または `users.id`） |
     | 氏名 | `users.name` |
     | 出勤日数 | 期間内で `clock_in_at` のある日数 |
     | 休日出勤日数 | パターン無し日への出勤で `substitute_for_date` が null の日数 |
     | 有給/特別休暇/欠勤日数 | `attendance_leaves` を区分別に日数集計 |
     | 就労時間 | Σ（給与用在社時間 − 休憩） |
     | 普通残業 | Σ（残業 − 深夜残業重複） |
     | 深夜残業 | Σ（全勤務 ∩ 22:00–翌5:00、休憩控除） |
     | 遅早回数 | Σ（ベース解決日で遅刻 or 早退があった日数。1日最大1回） |
     | 遅早時間 | Σ（遅刻分＋早退分） |
   - 時間は **HH:MM**（総分→`floor(min/60)`:`min%60` ゼロ埋め）、日数は小数2桁。文字コードは既存CSVに合わせる（SJIS/UTF-8 BOM 等は要確認）。
3. ルート：`GET /attendance/export-summary-csv` → `admin.attendance.export-summary-csv`。

### 変更（管理画面 Index.vue）
4. 既存「CSVダウンロード」付近に「月次集計CSV」ボタンを追加（現在のフィルタ期間を引き継いでダウンロード）。

## 実装順序（依存関係）
1. ①パターン上書き＋振替対象日（土台：ベース時刻・休日出勤・遅早が依存）
2. ②休暇入力機能（有給/特別休暇/欠勤の入力元）
3. ③月次集計CSV（①②を集計）

各段階で `php artisan migrate` → 動作確認 → 次段階。

## 検証（end-to-end）
- ①：会社カレンダーにパターン無しの日に打刻のあるレコードを用意 → Index がオレンジ強調 → モーダルでパターン選択＆振替対象日登録 →
  ベース時刻が算出表示・オレンジ解消、振替対象日ありで休日出勤に含まれないことを確認。
- ②：休暇登録モーダルで有給/特別休暇/欠勤を登録 → 一覧反映 → CSVの該当日数に反映。
- ③：テスト期間で月次集計CSVをダウンロードし、各列が手計算と一致することを確認（特に 就労時間・普通残業・深夜残業・遅早）。
- 回帰：既存の明細CSV・給与区分表示・通常日のベース時刻が不変であること。
- `php artisan test --filter Attendance`、`npm run build`。

## 留意点
- 深夜残業と普通残業の二重計上防止（普通残業＝残業−深夜残業重複）をテストで担保。
- 遅早はベース解決日のみ対象（パターン未設定で上書きもされていない日はカウントしない）。
- 社員番号は暫定（ユーザーID代用）。正式番号が必要になれば `users` にカラム追加で差し替え。
- 本番反映時は京呉服平田VPSの暫定ルール（`config:cache` 禁止・コンテナ削除禁止）に従い、`php artisan migrate` 要。
- CSVの文字コード・改行は既存 `exportCsv()` の実装に合わせる（実装前に確認）。
